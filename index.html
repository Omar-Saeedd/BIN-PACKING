📌 What is Bin Packing? ▼

Formal Definition

Input: Items with sizes s₁, s₂, …, sₙ ∈ (0, 1], integer k
Question: Can all n items fit into at most k bins, each with capacity 1?
Optimization version: Minimize the number of bins used.

Real-World Applications

  • Cutting stock: Minimize wasted material when cutting rolls
  • Container loading: Fewest containers to ship all cargo
  • Memory allocation: Fit processes into fixed-size memory pages
  • Cloud provisioning: Minimize VM instances to run all workloads
  • TV scheduling: Pack ads into broadcast slots

Why is it Hard?

The decision version of Bin Packing is NP-complete — no polynomial-time exact algorithm is known. The brute-force approach tries all possible assignments: for n items and a fixed k bins that's kⁿ combinations (for a fixed k). An optimizing brute-force tries k = 1, 2, …, n in sequence, so the total work is closer to 1ⁿ + 2ⁿ + … + OPTⁿ. For n=20 and k=5 that's already 95 trillion possibilities for that k alone. A practical approach is First-Fit Decreasing (FFD), a greedy approximation achieving at most (11/9)·OPT + 6/9 bins.

📚 Worked Example — FFD Algorithm ▼

Items: [0.5, 0.4, 0.3, 0.7, 0.2, 0.6]  |  Capacity = 1.0

Step 0 — Sort Decreasing: [0.7, 0.6, 0.5, 0.4, 0.3, 0.2]

Place 0.7: No bins open → open Bin 1. Place 0.7. Bin 1 remaining = 0.3.
Place 0.6: Bin 1 has 0.3 remaining — 0.6 doesn't fit. Open Bin 2. Place 0.6. Bin 2 remaining = 0.4.
Place 0.5: Bin 1: 0.3 < 0.5. Bin 2: 0.4 < 0.5. Open Bin 3. Place 0.5. Bin 3 remaining = 0.5.
Place 0.4: Bin 1: 0.3 < 0.4. Bin 2: 0.4 ≥ 0.4 ✓. Place in Bin 2. Remaining = 0.
Place 0.3: Bin 1: 0.3 ≥ 0.3 ✓. Place in Bin 1. Remaining = 0.
Place 0.2: Bin 1: 0 < 0.2. Bin 2: 0 < 0.2. Bin 3: 0.5 ≥ 0.2 ✓. Place in Bin 3. Done!
Result: Bin 1=[0.7, 0.3], Bin 2=[0.6, 0.4], Bin 3=[0.5, 0.2]. 3 bins = OPT ✓

FFD is not always optimal

Counterexample: Items [0.52, 0.44, 0.27, 0.27, 0.25, 0.23]. FFD sorts them as [0.52, 0.44, 0.27, 0.27, 0.25, 0.23]. It packs Bin 1=[0.52,0.44], Bin 2=[0.27,0.27,0.25], Bin 3=[0.23], so FFD uses 3 bins. However, the optimal packing is Bin 1=[0.52,0.25,0.23] and Bin 2=[0.44,0.27,0.27], so OPT = 2 bins. Therefore, FFD is suboptimal. The worst-case guarantee is (11/9)·OPT + 6/9 ≈ 1.22·OPT.

We prove Bin Packing is NP-complete by (1) showing it is in NP, and (2) reducing Partition to it in polynomial time.

✅ Step 1: Bin Packing ∈ NP ▼

Certificate

A proposed solution is an assignment A : {1,…,n} → {1,…,k} mapping each item i to a bin A(i). Size: O(n log k) bits — polynomial.

Polynomial-Time Verifier

Given assignment A:
1. Check there are at most k distinct bins — O(n)
2. For each bin b, compute Σ{sᵢ : A(i)=b} — O(n)
3. Check each bin sum ≤ capacity C — O(k)
Total: O(n) — polynomial ✓

🔗 Step 2: Reduction from Partition ▼

Partition Problem (source)

Input: A multiset S = {a₁, a₂, …, aₙ} of positive integers with sum 2T.
Question: Can S be partitioned into two subsets each summing to exactly T?

Construction (polynomial-time)

Given Partition instance S = {a₁,…,aₙ} with sum 2T:
Edge cases (automatic NO): If the total sum of S is odd, no equal partition can exist — the Partition instance is immediately NO. Similarly, if any aᵢ > T, no subset containing aᵢ can sum to exactly T — also automatically NO. In both cases the reduction still produces a valid Bin Packing instance, but one that is trivially unsolvable with k=2 bins.
1. Otherwise, normalize each item: let sᵢ = aᵢ / T, so each sᵢ ∈ (0, 1].
2. Set bin capacity C = 1 (normalized)
3. Set k = 2 bins
Construction time: O(n) divisions — polynomial ✓

Key insight: After normalization (if valid), Σsᵢ = 2, which exactly fills 2 bins of capacity 1. The items fit into 2 bins if and only if they can be split into two groups each summing to exactly 1, which corresponds to a valid Partition.

Correctness Proof

Forward (⟹): Partition YES ⟹ Bin Packing YES

Suppose S has a partition into subsets S₁, S₂ with Σ(S₁) = Σ(S₂) = T.
Normalize: for items in S₁, Σ(sᵢ : i∈S₁) = T/T = 1. Same for S₂.
Pack S₁ into Bin 1, S₂ into Bin 2. Each bin has total = 1 = capacity.
∴ All items fit in 2 bins. Bin Packing is YES. ✓

Backward (⟸): Bin Packing YES ⟹ Partition YES

Suppose all n items fit in 2 bins of capacity 1.
Let S₁, S₂ be the items in Bin 1, Bin 2 respectively.
Since total = 2 and both bins have capacity 1:
Σ(sᵢ : i∈S₁) ≤ 1 and Σ(sᵢ : i∈S₂) ≤ 1, and both sum to 2.
∴ Each bin's sum = exactly 1, so Σ(aᵢ : i∈Sⱼ) = T for j=1,2.
∴ S is partitioned. Partition is YES. ✓

Conclusion

The reduction runs in O(n) time (polynomial). S has an equal partition iff the normalized items fit in 2 unit bins. Since Partition is NP-complete, Bin Packing is NP-hard. Combined with Step 1 (Bin Packing ∈ NP), Bin Packing is NP-complete. □

🧮 Approximation Ratio Analysis ▼

FFD Theorem (Johnson 1973, refined by Yue 1991)

FFD uses at most (11/9)·OPT + 6/9 bins. This is tight — Johnson showed a family of instances where FFD uses exactly (11/9)·OPT bins. FFD gives a simple practical guarantee, but more advanced approximation algorithms exist with better bounds.

Why Sorting Helps

Placing large items first forces them into bins early, when bins are still empty and can accommodate them. This leaves smaller items to fill the remaining gaps flexibly. Without sorting (First-Fit), large items may arrive late and require new bins, wasting earlier gaps.
